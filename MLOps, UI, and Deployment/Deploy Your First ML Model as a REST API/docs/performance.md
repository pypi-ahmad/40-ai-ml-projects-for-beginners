# Performance Validation (Measured)

Source: `artifacts/performance/api_performance_summary.json` generated by
`python scripts/benchmark_api.py --in-process --n-single 80 --n-batch 30 --batch-size 64`.

| Metric | Value |
|---|---:|
| Cold start latency | 1846.04 ms |
| Warm avg latency (single) | 20.01 ms |
| Warm p50 latency (single) | 18.48 ms |
| Warm p95 latency (single) | 28.12 ms |
| Throughput (single) | 49.97 req/s |
| Batch size | 64 |
| Batch avg latency | 19.51 ms |
| Batch p95 latency | 31.31 ms |
| Batch throughput | 3280.36 records/s |
| RSS before benchmark | 105.80 MB |
| RSS after benchmark | 204.05 MB |
| RSS delta | 98.25 MB |

## Interpretation

- Cold start is dominated by model loading and first-time SHAP/model imports.
- Warm single prediction latency is low enough for typical synchronous API usage.
- Batch mode significantly improves throughput vs repeated single requests.
- Memory increases after warm-up due loaded model + inference runtime state.

## Caveat

Network-restricted sandboxes can block loopback probes (`ConnectError: [Errno 1] Operation not permitted`).
In those environments, use `--in-process` benchmark mode and rely on startup logs for uvicorn boot verification.
