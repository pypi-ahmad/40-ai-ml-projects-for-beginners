# Production-Grade Multimodal AI Platform

Production-ready local multimodal platform with FastAPI, Streamlit, Typer CLI, ChromaDB, SQLite, OCR pipelines, multimodal RAG, and MCP tooling.

This README is based on a real terminal execution done on **June 30, 2026** in this repository. All commands, artifacts, and verification outputs below come from actual runs.

## 1. What This Project Delivers

The platform supports multimodal workflows for images and documents:

- Caption generation (short, detailed, social, technical, ALT-style).
- OCR for image/document inputs with native parse + OCR fallback.
- Embedding generation (text/image) and semantic retrieval via ChromaDB.
- Visual question answering and document-grounded Q&A (RAG pipeline).
- Image comparison and similarity scoring.
- Consolidated analysis endpoint (caption + detection + document summary pipeline).
- API, UI, CLI, and MCP server entry points sharing one service layer.

## 3. Architecture (How It Works)

### Core layers

1. Interfaces
- FastAPI: `src/multimodal_ai/api/`
- Streamlit: `src/multimodal_ai/ui/streamlit_app.py`
- CLI: `src/multimodal_ai/cli/main.py`
- MCP server: `src/multimodal_ai/mcp/server.py`

2. Service orchestration
- `PlatformService` in `src/multimodal_ai/services/platform_service.py`
- Unified business logic consumed by API/UI/CLI/MCP

3. Pipelines
- `DocumentPipeline` (native parse + OCR fallback)
- `RetrievalPipeline` (index/search)
- `MultimodalRAGPipeline` (ingest -> chunk -> retrieve -> answer)

4. Adapter/plugin registry
- Vision adapters
- OCR adapters
- LLM adapters
- Embedding adapters
- Detection adapters

5. Storage
- SQLite (metadata/history/usage): `data/multimodal.db`
- ChromaDB (vector collections): `data/chroma/chroma.sqlite3`

## 10. What Was Fixed During Finalization

Real errors found and fixed in this final run:

1. Build failure in offline-isolated `python -m build`
- Cause: isolated build env could not fetch `wheel`
- Fix: added dev dependency `wheel` and used `--no-isolation` for local build reproducibility

2. API `/vqa` runtime 500 during live e2e
- Cause: Ollama backend unavailable -> uncaught HTTP error in `OllamaLLMAdapter.complete`
- Fix: added resilient fallback return path in `src/multimodal_ai/adapters/llm.py`
- Result: `/vqa` now returns successful structured fallback response instead of crashing.
